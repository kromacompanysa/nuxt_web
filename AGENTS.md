# AGENTS.md

## Dev Commands

```bash
npm run dev      # Start dev server
npm run build    # Production build
npm run generate # Static site generation (SSG)
npm run preview  # Preview production build
npm run lint     # ESLint
npm run lint:fix # ESLint with auto-fix
npm run typecheck # vue-tsc type check
npm run postinstall # Runs nuxt prepare (regenerates .nuxt/)
```

## CI / Deploy

- Push to `main` triggers GitHub Actions
- Uses `npm` (not pnpm): `npm install && npm run generate`
- Output: `.output/public/` synced to S3 bucket + CloudFront invalidation

## Architecture

- **Nuxt 4**, `srcDir: "app/"` — all source code lives under `app/`
- **Components** auto-imported with path prefix:
  - `app/components/Default/NavBar.vue` → `<DefaultNavBar />`
  - `app/components/Portal/Footer.vue` → `<PortalFooter />`
- **`app/app.config.ts`** — central config for navbar menu structure, gallery images, and contact info (WhatsApp, email, map)
- **`.nuxt/tsconfig.*.json`** are generated by Nuxt — run `nuxt prepare` after modifying `nuxt.config.ts`

## Toolchain

- **Tailwind CSS 4** via `@tailwindcss/vite` plugin (not the PostCSS setup)
- **Swiper** deps pre-bundled in Vite (`optimizeDeps.include`)
- **Static output** (`nitro.preset: "static"`) goes to `.output/public/`

## Style

- Prettier: 100 char width, single attribute per line, Vue scripts/styles indented
- ESLint: `vue/multi-word-component-names` disabled (single-word components allowed)

## Key Files

- `nuxt.config.ts` — framework config (srcDir, Tailwind, devtools, SSG preset)
- `app/app.config.ts` — site-wide content config (nav, gallery, contact)
- `app/assets/css/main.css` — global styles (Tailwind entry)
- `eslint.config.ts` — ESLint config extending `.nuxt/eslint.config.mjs`
